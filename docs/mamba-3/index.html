1. The Compounding Error of ZOH

In Mamba-1 and Mamba-2, the continuous differential equation was discretized using Zero-Order Hold (ZOH). ZOH assumes that the input signal \(x(t)\) remains perfectly constant for the entire duration of the time step \(\Delta\).

Mathematically, this introduces an integration error margin of \(O(\Delta)\). While acceptable for short sequences, when processing contexts of 1M+ tokens, this approximation error compounds. The model's internal state slowly drifts away from the true continuous representation guaranteed by the HiPPO matrix.

2. Trapezoidal Recurrence (Crank-Nicolson)

Mamba-3 solves this drift by upgrading the integration scheme to the Trapezoidal Rule (Crank-Nicolson method). Instead of assuming the data is a flat stair-step, the model evaluates the state at both \(t-1\) and \(t\), essentially drawing a sloped line between the two points.

\[ h_t = \exp(\Delta_t A)h_{t-1} + \frac{\Delta_t}{2}\left(B_t x_t + \exp(\Delta_t A)B_{t-1}x_{t-1}\right) \]

By taking this weighted average, the approximation error drops from \(O(\Delta)\) down to \(O(\Delta^2)\). This drastically improves the model's ability to maintain high-fidelity memory over massive context windows.

3. Complex Geometric States (RoPE)

In earlier generations, the hidden state \(h_t\) consisted of real numbers (\(\mathbb{R}^N\)). Real exponential dynamics are excellent at one thing: controlling magnitude. A state dimension can decay slowly, decay quickly, or become unstable if the spectrum is wrong. But magnitude alone is a limited language for long context. It tells the model how much signal remains, not where that signal sits in a repeating or oscillatory pattern.

Mamba-3 adds a geometric degree of freedom by letting part of the state behave like a complex plane. A RoPE-style rotation changes the phase of the state without needing to erase its magnitude. That gives the recurrence a way to carry position-like structure through time while still using the SSM machinery for controlled decay and selection.

Complex state intuition decay controls radius, rotation controls phase
Mamba-3 makes the state less like a fading scalar and more like controlled geometry.

4. MIMO Rank Expansion

A standard Mamba block operates as a Single-Input Single-Output (SISO) system. If the model has 64 channels, it runs 64 entirely independent state spaces. The channels never share information. If you want to increase capacity and allow the channels to mix, the parameter count balloons quadratically \(O(D^2)\).

Mamba-3 introduces Multi-Input Multi-Output (MIMO) routing. The useful way to think about it is not "make everything dense." It is "give the recurrence a narrow mixing room." Channels enter a low-rank bottleneck, interact there, and then expand back out to the model heads.

MIMO rank expansion mix cheaply, then route back out
MIMO makes the state more expressive by mixing channels through a controlled low-rank path.

In practice, this matters because language features rarely live in one isolated channel. A name, a tense marker, or a long-range dependency may need several channels to coordinate. MIMO gives the recurrent state a structured way to combine those signals, improving capacity while keeping the parameter and compute budget disciplined.

So where does this leave SSMs?

The Mamba lineage is not simply "RNNs are back." It is a sequence of answers to the same pressure: Transformers are easy to parallelize but expensive to remember with, while recurrent models remember cheaply but are hard to train at scale. Mamba tries to keep the recurrent memory and recover the hardware shape.

Mamba-1 Proves that a recurrent state can be selective enough for language instead of being a blind compression.
Mamba-2 Shows that recurrence can be reorganized into matrix-friendly chunks without giving up linear-time memory.
Mamba-3 Pushes the state itself to become more faithful, geometric, and expressive across channels.

That is the real story: not attention versus recurrence as a slogan, but a search for architectures that can decide what matters, store it compactly, and still run like they belong on modern accelerators.
