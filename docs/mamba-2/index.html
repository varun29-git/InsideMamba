1. The Two Warring Paradigms

For years, sequence modeling was split into two camps. On one side were Recurrent Models (RNNs and SSMs). They process data linearly, updating a hidden state \(h_t\) at each step. Because they only need to look at the current state to predict the next word, they are incredibly fast during generation (inference). However, because step \(t\) relies on step \(t-1\), they are fundamentally sequential. You cannot train them in parallel, making them excruciatingly slow to scale.

On the other side was Attention (Transformers). Attention ignores sequence entirely; it looks at all tokens simultaneously through massive matrix multiplications. This means it trains blazing fast on GPUs. But during generation, it must constantly re-read the entire history, creating a catastrophic \(O(N^2)\) memory bottleneck.

2. The Duality Proof

The core breakthrough of Mamba-2 is a mathematical proof showing that these two seemingly opposite paradigms are actually the exact same thing.

Standard Attention computes an output sequence by multiplying Queries and Keys with a causal attention mask \(L\):

\[ Y = (Q K^T \odot L) V \]

In a normal Transformer, \(L\) is just a lower-triangular matrix of ones (so tokens only look backward). But what happens if we structure \(L\) differently? The Mamba-2 authors proved that if \(L\) is constructed as a 1-semiseparable matrix, the Attention equation mathematically collapses into the linear SSM recurrence.

3. The 1-Semiseparable Matrix

A 1-semiseparable matrix is a matrix where every element below the diagonal can be factored into a sequence of scalar multipliers. In the context of Mamba, it represents exponential state decay over distance:

\[ L_{ij} = \prod_{k=j+1}^i a_k \quad \text{for } i \ge j \]

If \(a_k\) is the decay parameter from the SSM, the matrix \(L\) perfectly encodes how much of token \(j\)'s information survives until time step \(i\). By multiplying \((Q K^T)\) by this specific decay matrix \(L\), we are computing an SSM—but doing so using the matrix multiplication of Attention.

4. The Hardware Masterstroke: Block Decomposition

Why is this duality so important? Because GPUs are designed to do matrix multiplication using Tensor Cores, but they are terrible at sequential loops.

Armed with the duality proof, Mamba-2 uses an algorithm called State Space Duality (SSD) to compute the sequence via Block Decomposition. The sequence is sliced into chunks (e.g., blocks of 64 tokens).

Inside each chunk (the diagonal blocks of the matrix), the interactions between tokens are computed entirely in parallel using Dual Attention matrix multiplication. Then, the single accumulated "boundary state" at the end of the chunk is sequentially passed to the next block (the off-diagonal blocks).

This avoids materializing the massive intermediate states in GPU memory, fully unleashing the power of Tensor Cores and resulting in a 2x to 4x training speedup over Mamba-1.
