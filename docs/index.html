Chronicles of State Space

How to process infinite context
in linear time.

Imagine trying to read a book, but you can only remember the last page. Traditional State Space Models do exactly this. They compress history blindly. The Mamba architecture changes the rules by selectively choosing what to "Remember" and what to "Forget" at every step.

What this site is

A guided walk through the mechanics, not a paper summary.

InsideMamba follows the architecture as it evolves: first selection, then duality, then better continuous-time dynamics. Each generation is framed around the specific limitation it fixes, so the math has a reason to exist before it appears.

Memory is compressed. An SSM does not keep every token. It keeps a state, and the quality of that state determines what survives.
Selection makes it useful. Mamba lets the current token choose how much to write, read, and forget instead of using one fixed recurrence.
Structure makes it fast. Mamba-2 shows that recurrence can be reorganized into GPU-friendly matrix blocks without abandoning linear-time state.
The route

Three generations, one question: what should memory keep?

Generation I Selective Scan The model learns when to freeze memory, when to overwrite it, and when to read from it. Generation II State Space Duality The recurrence is re-expressed as structured masked matrix multiplication over chunks. Generation III Trapezoidal MIMO The state update becomes more faithful to continuous dynamics and less isolated across channels.
