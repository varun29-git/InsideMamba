1. The Continuous Origin

Unlike Transformers, which were born from discrete sequence-to-sequence translation, State Space Models (SSMs) are rooted in continuous-time control theory. The foundation of Mamba is a simple differential equation that maps a 1-dimensional input signal \(x(t)\) to an \(N\)-dimensional latent state \(h(t)\), and then projects it to an output \(y(t)\):

\[ h'(t) = A h(t) + B x(t) \] \[ y(t) = C h(t) \]

The matrix \(A\) is the most critical component. If initialized randomly, the model forgets past context instantly. Mamba initializes \(A\) using HiPPO (High-order Polynomial Projection Operators). The HiPPO matrix mathematically guarantees that the state \(h(t)\) maintains an optimal compression of the entire history of the input sequence using orthogonal polynomials.

2. The Digital Bridge: Discretization

Language models process discrete tokens, not continuous waveforms. To bridge this gap, the continuous system is discretized using a step size \(\Delta\).

Using the Zero-Order Hold (ZOH) rule, the continuous matrices \(A\) and \(B\) are transformed into their discrete counterparts \(\bar{A}\) and \(\bar{B}\):

\[ \bar{A} = \exp(\Delta A) \] \[ \bar{B} = (\Delta A)^{-1} (\exp(\Delta A) - I) \cdot \Delta B \]

This transforms the differential equation into a simple recurrent loop: \( h_t = \bar{A} h_{t-1} + \bar{B} x_t \).

3. The Flaw of LTI and the Convolutional Hack

In early SSMs (like S4), the matrices \(\bar{A}\) and \(\bar{B}\) were Linear Time-Invariant (LTI). They were fixed numbers. Because the parameters never changed, the model could not filter information. It treated a critical keyword and a comma with the exact same weight.

However, LTI models had a massive advantage: because the recurrence was constant, it could be mathematically unrolled into a single Convolution. By applying a Fast Fourier Transform (FFT), the model could process the entire sequence simultaneously during training, making it as fast as a Transformer.

4. The S6 Breakthrough: Selection

Mamba sacrifices the FFT convolution to achieve intelligent, content-aware filtering. It makes the parameters dependent on the current input token \(x_t\):

\[ B_t = W_B x_t \quad C_t = W_C x_t \quad \Delta_t = \text{softplus}(\text{Parameter} + W_\Delta x_t) \]

This breaks the LTI constraint. The model can no longer use convolutions. But it gains the ability to reason.

The \(\Delta_t\) Forget Gate: If the model reads a useless token, the neural network outputs a tiny \(\Delta_t\). Looking back at the discretization formula, if \(\Delta \to 0\), then \(\bar{A} = \exp(0) = I\) (the Identity matrix). The state \(h_t = I h_{t-1} + 0\). The model literally freezes its memory and ignores the token. Conversely, a large \(\Delta_t\) drives \(\bar{A} \to 0\), flushing old memory to absorb new, high-priority information.

5. The Hardware-Aware Parallel Scan

By breaking the LTI constraint, Mamba lost the ability to use fast FFT convolutions. If Mamba had to process a 100,000-token sequence using a standard PyTorch `for` loop, it would be unusably slow due to the Memory Wall.

GPUs have massive compute capabilities (Tensor Cores) but very slow memory bandwidth (HBM). Materializing the massive \(N\)-dimensional state \(h_t\) at every step and saving it to HBM takes \(O(BLDN)\) memory reads/writes.

To solve this, Mamba implements a custom Hardware-Aware Fused Kernel. It loads the sequence into the GPU's ultra-fast SRAM cache, computes the entire recurrence using a Work-Efficient Parallel Prefix Scan algorithm, and only writes the final output \(y_t\) back to the slow HBM. This engineering masterstroke is what allows Mamba to train as fast as a Transformer while reasoning sequentially.
